Api para macros do Notes


Este documento descreve as funções que uma macro do Notes pode usar para se comunicar com o Notes e o sistema operacional, podendo assim acessar arquivos, manipular o editor, etc.

O Notes implementa 5 objetos que você pode usar em seus scripts: "notes", "editor", "tabs", "dlgs" e "os". Todas as funções destes objetos estão descritas abaixo. Após a descrição destes objetos, não descritas as ações e propriedades do Notes e de seus painéis que você pode usar em seus scripts (elas podem ser passada em funções do objeto "notes").


O objeto "notes"

O objeto "notes" permite que a macro se comunique com o Notes, peça informações para ele e execute ações do Notes.

notes.getNotesDir()
Retorna o diretório do Notes

notes.getMacrosDir()
Retorna o diretório das macros

notes.getSettingsDir()
Retorna o diretório de configurações do Notes

notes.getActiveProfile()
Retorna o profile atual

notes.getDefaultFileType()
Retorna o tipo de arquivo padrão

notes.exec(ActionName)
Executa a ação passada
Parâmetros:
ActionName - nome da ação a ser executada. Veja a lista de ações no final desta referência.

notes.update()
Força o Notes a atualizar. Importante apenas dentro de algum loop muito demorado.

notes.panelSetProperty(PanelName, Property, Value)
Seta uma propriedade de um painel do Notes
Parâmetros:
PanelName - O nome do painel
Property - Nome da propriedade que se quer alterar
Value - Valor que será assinalado a propriedade

notes.panelGetProperty(PanelName, Property)
Retorna o valor de uma propriedade de um painel do Notes
Parâmetros:
PanelName - O nome do painel
Property - Nome da propriedade de que se quer saber o valor

notes.panelExec(PanelName, ActionName, Param)
Executa uma ação de um painel
Parâmetros:
PanelName - O nome do painel
ActionName - Nome da ação a executar
Param - parâmetro passado para a ação

notes.status(Text)
Coloca o texto na statusbar do Notes
Parâmetros:
Text - o texto a ser mostrado na StatusBar



Objeto "editor"

O objeto editor permite que você manipule o editor do Notes. Por padrão as funções manipulam o editor que estiver ativo. Elas podem também manipular os editores que não estão ativos se você passar o índice do editor que você quer manipular no último parâmetro. Por exemplo, se você quiser manipular o primeiro editor, passe 0 como último parâmetro.

editor.insert(Text, [EditorIndex])
Insere texto na posição atual do editor. Versão mais rápida e menos segura de setSelText
Parâmetros:
Text - O texo a ser inserido
EditorIndex (opcional) - índice do editor a ser manipulado

editor.getText([EditorIndex])
Retorna o texto do editor
Parâmetros:
EditorIndex (opcional) - índice do editor a ser manipulado

editor.setText([EditorIndex])
Seta o texto do editor
Parâmetros:
EditorIndex (opcional) - índice do editor a ser manipulado

editor.getSelText([EditorIndex])
Retorna o texto selecionado
Parâmetros:
EditorIndex (opcional) - índice do editor a ser manipulado

editor.setSelText([EditorIndex])
Seta o texto selecionado
Parâmetros:
EditorIndex (opcional) - índice do editor a ser manipulado

editor.getLineText([Line][,EditorIndex])
Retorna o conteúdo da linha. Se nenhum parâmentro é passado, retorna o conteúdo da linha atual
Parâmetros:
Line (opcional) - Linha a ser manipulada
EditorIndex (opcional) - índice do editor a ser manipulado

editor.setLineText(Text [, Line][, EditorIndex])
Permite modificar o conteúdo da linha
Parâmetros:
Line (opcional) - Linha a ser manipulada
EditorIndex (opcional) - índice do editor a ser manipulado

editor.getLine([EditorIndex])
Retorna o número da linha atual

EditorIndex (opcional) - índice do editor a ser manipulado

editor.setLine(Value [, EditorIndex])
Permite modificar o número da linha atual
Parâmetros:

EditorIndex (opcional) - índice do editor a ser manipulado

editor.getCol([EditorIndex])
Retorna a linha atual
Parâmetros:
Col - Modifica a coluna atual
EditorIndex (opcional) - índice do editor a ser manipulado
EditorIndex (opcional) - índice do editor a ser manipulado

editor.getSelStart([EditorIndex])

EditorIndex (opcional) - índice do editor a ser manipulado

editor.setSelStart(Value [, EditorIndex])

editor.getSelLength([EditorIndex])

Parâmetros:
EditorIndex (opcional) - índice do editor a ser manipulado

editor.setSelLength(Value [, EditorIndex])

editor.getLinesCount([EditorIndex])

Parâmetros:
EditorIndex (opcional) - índice do editor a ser manipulado

editor.geFileType([EditorIndex])
Retorna o tipo de arquivo do editor
Parâmetros:
EditorIndex (opcional) - índice do editor a ser manipulado

editor.getTabSize([EditorIndex])
Retorna o tamanho do caracter tab
Parâmetros:
EditorIndex (opcional) - índice do editor a ser manipulado

editor.setTabSize(Value [, EditorIndex])
Permite modificar o tamanho do caracter tab
Parâmetros:
Value - novo valor
EditorIndex (opcional) - índice do editor a ser manipulado

editor.getIndentSize([EditorIndex])
Retorna a quantidade de caracters espaços usados para identar o código
Parâmetros:
EditorIndex (opcional) - índice do editor a ser manipulado

editor.setIndentSize(Value [, EditorIndex])
Permite modificar a quantidade de caracters espaços usados para identar o código
Parâmetros:
Value - novo valor
EditorIndex (opcional) - índice do editor a ser manipulado

editor.getModified([EditorIndex])
Retorna true se o texto tiver sido modificado
Parâmetros:
EditorIndex (opcional) - índice do editor a ser manipulado

editor.setModified(Value [, EditorIndex])
Permite mudar o status de modificado do texto
Parâmetros:
Value - novo valor
EditorIndex (opcional) - índice do editor a ser manipulado

editor.getReadOnly([EditorIndex])
Retorna true se o editor for read-only
Parâmetros:
EditorIndex (opcional) - índice do editor a ser manipulado

editor.setReadOnly(Value [, EditorIndex])
Permite modiicar o status de leitura/escrita do editor
Parâmetros:
Value - novo valor
EditorIndex (opcional) - índice do editor a ser manipulado

editor.lockUpdates()
Faz o editor parar de atualizar. Você deve chamar "update" posteriormente ou o editor não voltará a ser pintado.

editor.update()
Atualiza o editor ou volta a atualizá-lo após um lockUpdates

editor.inserLineBreak([EditorIndex]);
Insere uma quebra de linha

editor.getMark(Index, [, EditorIndex])

editor.getMarksCount([EditorIndex]))

Remove a marca da linha passada

editor.clearMarks([EditorIndex]);
Limpa todos os marcadores do editor
Parâmetros:
EditorIndex (opcional) - índice do editor a ser manipulado



O objeto "tabs"

O objeto tabs permite manipular as tabs (abas) do Notes. Usando ele você pode abrir arquivos, criar novos arquivos, mudar o documento (tab) ativo, etc.

tabs.nextTab()
Vai para o próximo documento

tabs.previusTab()
Vai para o documento anterior

tabs.gotoFile(FileName)
Vai para o arquivo especificado
Parâmetros:
FileName - nome do arquivo

tabs.gotoFileAndMarkLine(fileName, line)
Vai para o arquivo especificado e marca a linha em vermelho
Parâmetros:
FileName - nome do arquivo
Line - linha a ser marcada

tabs.getTabIndex()
Retorna o índice da tab atual

tabs.setTabIndex(Value)
Permite modificar qual tab está ativa

tabs.getTabCaption([TabIndex])
Pega o título da tab
Parâmetros:
EditorIndex (opcional) - índice do tab a ser manipulada

tabs.getFileName([TabIndex])
Retorna o nome do arquivo, incluindo o caminho completo para ele
Parâmetros:
EditorIndex (opcional) - índice do tab a ser manipulada

tabs.openFile(fileName);
Abre o arquivo especificado
Parâmetros:
FileName - arquivo que deverá ser aberto

tabs.getCount()
retorna "startpage, newfile, file, none"
Parâmetros:
EditorIndex (opcional) - índice do editor a ser manipulado



O objeto "os"

O objeto os permite a macro se comunicar com o sistema operacional. Através dele é possível manipular arquivos, as variáveis do ambiente, etc.

os.exec(Application, Args)
Executa um programa
Parâmetros:
EditorIndex (opcional) - índice do editor a ser manipulado
Application - programa a ser executado
Args - argumentos que devem ser passados ao programa

os.execAndWait(Application, Args)
Executa um programa e aguarda ele terminar. Retorna o exit code do processo.
Parâmetros:
Application - programa a ser executado
Args - argumentos que devem ser passados ao programa

os.getClipboard()
Retorna o conteúdo da clipboard (área de transferência)

os.setClipboard(Value)
Seta o conteúdo da clipboard
Parâmetros:
Value - novo valor

os.getCurDir()
Retorna o diretório atual do sistema operacional

os.setCurDir(Value)
Permite modificar o diretório atual
Parâmetros:
Value - novo diretório

os.setEnv(name, value)
Modifica ou cria uma variável de ambiente
Parâmetros:
name - nome da variável
Value - novo valor

os.getEnv(name)
Retonra o valor de uma variávle de ambiente
Parâmetros:
name - nome da variável

os.getDirDelimiter()
Retorna o delimitador de diretórios do sistema operacional. Exemplo: "\" no windows e "/" no linux, BSD, etc.

os.getName()
Retorna o nome do sistema operacional

os.getUser()
Retorna o nome do usuário

os.getVersion()
Retorna a versão do sistema operacional

os.fileGetContents(filename)
Retorna o conteúdo de um arquivo
Parâmetros:
FileName - nome do arquivo

os.filePutContents(filename, content)
Modifica o conteúdo de um arquivo/cria um arquivo
Parâmetros:
FileName - nome do arquivo
Content - conteúdo que será salvo no arquivo

os.fileAppend(filename, content)
Adiciona uma string ao conteúdo de um arquivo
Parâmetros:
FileName - nome do arquivo
Content - conteúdo que será salvo no arquivo

os.fileCopy(fromfile, tofile [, overwrite = true]) bool
Copia um arquivo
Parâmetros:
fromfile - arquivo a ser copiado
tofile - arquivo para onde o arquivo orginal deve ser copiado
overwrite (opcional) - diz se a função pode sobrescrever arquivos durante o processo

os.fileDelete(filename)
Deleta um arquivo
Parâmetros:
FileName - nome do arquivo

os.fileSearch(dirname [, mask = '*'] [, recursive = false])
Busca um arquivo. Retorna uma string com os nomes dos arquivos separados por uma quebra de linha. Voce pode usar "os.fileSearch().split('\n')" para pegar o resultado como um array.
Parâmetros:
DirName - nome do diretório onde a pesquisa deverá ser feita
Mask (opcional) - máscara de busca, ex: "*.cpp"
Recursive (opcional) - se a busca deve ser recursiva

os.fileIsReadOnly(filename)
Retorna true se o arquivo for readonly
Parâmetros:
FileName - nome do arquivo

os.fileSize(filename)
Retorna o tamanho do arquivo
Parâmetros:
FileName - nome do arquivo

os.fileExists(filename)
Retorna true se o arquivo existir
Parâmetros:
FileName - nome do arquivo

os.dirExists(dirname)
Retorna true se o diretório existir
Parâmetros:
DirName - nome do diretório

os.dirCopy(fromdir, todir [, mask = '*' ][, overwrite = true])
Copia um diretório
Parâmetros:
fromdir - diretório a ser copiado
todir - diretório para onde o diretório orginal deve ser copiado
Mask (opcional) - máscara de busca, ex: "*.cpp"
overwrite (opcional) - diz se a função pode sobrescrever arquivos durante o processo

os.dirDelTree(dirname [, mask = '*' ])
Deleta uma árvore de diretórios
Parâmetros:
DirName - nome do diretório
Mask (opcional) - máscara de busca, ex: "*.cpp"

os.dirDelete(dirname)
Deleta um diretório vazio
Parâmetros:
DirName - nome do diretório

os.dirCreate(dirname [, force])
Cria um diretório
Parâmetros:
DirName - nome do diretório
force - se true, a função tentará criar todos os diretórios necessários para criar o caminho pedido

os.move(name)
Move um arquivo
Parâmetros:
Name - nome do arquivo/diretório

os.rename(name)
Renomeia um arquivo
Parâmetros:
Name - nome do arquivo/diretório



O objeto "dlgs"

O objeto dlgs permite chamar diálogos para que a macro possa interagir com o usuário. Até mesmo um diálogo para fazer IntelliSense está disponível.

alert(msg)
Mostra uma mensagem para o usuário
Parâmetros:
msg - mensagem

prompt(title[, value])
Mostra um diálogo que permite ao usuário entrar uma string
Parâmetros:
value (opcional) - valor incial
title - título do diálogo

dlgs.message(msg)
Mostra uma mensagem para o usuário
Parâmetros:
msg - mensagem

dlgs.confirm(msg)
Mostra uma mensagem para o usuário de confirmação, usada antes de operações que o usuário possa querer cancelar. Retorna "yes", "no" ou "cancel"
Parâmetros:
msg - mensagem

dlgs.question(msg)
Mostra uma pergunta ao usuário. Retorna "yes" ou "no"
Parâmetros:
msg - mensagem

dlgs.error(msg)
Mostra uma mensagem de erro para o usuário
Parâmetros:
msg - mensagem

dlgs.input(title[, value])
Mostra um diálogo que permite ao usuário entrar uma string. Retorna a string preenchida pelo usuário.
Parâmetros:
value (opcional) - valor incial
title - título do diálogo

dlgs.fileOpen()
Permite ao usuário selecionar um arquivo

dlgs.fileSave()
Permite ao usuário entrar o nome de um arquivo que ainda não existe e pergunta se o usuário quer substituir o arquivo no caso do usuário entra o caminho para um arquivo existente.

dlgs.folder([title])
Permite ao usuário selecionar uma pasta
Parâmetros:
title - título do diálogo

dlgs.color()
Permite ao usuário selecionar uma cor. Retorna a cor no formato hexadecimal, como usando em HTML, CSS, etc.

dlgs.list(title, values [, quickfind])
Permite ao usuário selecionar uma das entras de uma lista. Retorna o item selecionado
Parâmetros:
title - título do diálogo
Values - deve conter uma lista de strings separadas por quebras de linha
QuickFind - se true, mostra um campo que permite selecionar a entrada rapidamente

dlgs.codeCompletition(VisibleValues [, HiddenValues])
Mostra um listbox para fazer CodeCompletition/Intellisense. Depois de chamado o diálogo será mostrado ao usuário até ele escolher um item ou cancelar a operação. A função retornará logo que o diálogo for mostrado, isto é, não aguardará o usuário selecionar um item ou cancelar a operação. Se o usuário escolher um item, ele será inserido no editor. Se tiverem sido passado valores no parâmetro HiddenValues, o valor de HiddenValues será inserido no lugar do valor correspondente de VisibleValues.
Parâmetros:
VisibleValues - valores que serão mostrados ao usuário. Devem estar separados por quebras de linha.
HiddenValues - valores que serão inseridos se existirem no lugar do valor escolhido pelo usuário.

Painel biblioteca ("Library")

Uma macro pode se comunicar o painel biblioteca e outro painéis usando as funções "notes.panelExec", "notes.panelGetProperty" e "notes.panelSetProperty". O nome do painel deve ser passado como primeiro parâmetro. Ex: "notes.panelExec('Library', 'expand', '')" expande a árvore de items da biblioteca.

(ação) expand
Expande todas as categorias da biblioteca.

(ação) collapse
Fecha todas as categorias da biblioteca.

(ação) insert
Insere o item que estiver selecionado na biblioteca.

(propriedade)FileType
Permite ler e modificar o tipo de arquivo que está sendo mostrado na biblitoeca do Notes.



Painel saída ('Output')

Você pode chamar as ações e acessar as propriedades do painel saída usando o nome "Output". O painel saída permite que você rode aplicativos e captura a saída no painel ou no editor. Você pode também manipular o conteúdo do painel e modificar várias opções. Abaixo está uma lista com as ações e propriedades deste painel:

(ação) run
Executa o arquivo especificado em "FileToRun" com as configurações atuais.

(ação) getoutput
Retorna o conteúdo do painel saída.

(ação) add
Adiciona a string passada como parâmetro a saída.

(ação) scanproblems
Faz o painel buscar erros (de compilação) na saída e reportar eles ao Notes. O painel faz isto automaticamente quando "AutoScanForProblems" está ativado.

(ação) clear
Limpa o conteúdo do painel.

(ação) stop
Tenta parar o programa que esta sendo executado.

(propriedade) Running
Permite saber se o painel está rodando algum programa e capturando a saída.

(propriedade) CaptureTo
Especifica para onde a saída do arquivo executado deve ser capturada. Usando "outputwindow" captura no próprio painel de saída. Se setado para "editor" captura no editor.

(propriedade) FileToRun
Arquivo que o painel deve rodar e capturar a saída.

(propriedade) Arguments
Argumentos que serão passados ao arquivo especificado em "FileToRun" quando ele for executado (isto é, quando a ação "run" for chamada).

(propriedade) WorkDir
Permite setar o diretório de trabalho do sistema operacional antes de executar o arquivo especificado em "FileToRun".

(propriedade) ErrorsAutoParse
Indica se o parser automático de erros de compilação deve ser usado. Se setado para false, tentará usar a expressão regular setada em "ErrorsRegexParser" para entender a saída.

(propriedade) ErrorsReFilePos
Posição em que a expressão regular retornará o nome do arquivo com erro.

(propriedade) ErrorsReLinePos
Posição em que a expressão regular retornará a linha.

(propriedade) ErrorsValidFiles
String como uma lista de arquivos em cada linha. Usada quando arquivos com caminhos relativos e/ou incompletos são reportados como tendo erros pelo compilador para completar o nome do arquivo. Em geral setado para os arquivos abertoso no Notes e os arquivos do projeto atual.

(propriedade) AutoScanForProblems
Se setado para true, busca por erros na saída e preenche a lista de problemas (aquela que mostra os ícones de erro na gutter) com o que encontrar.



Painel de verficação ('Verification')

Você pode chamar as ações e acessar as propriedades do painel de verificação usando o nome "Verification". O painel de verificação usa o HTMLTidy para corrigir e verifica código HTML, XML e XHTML. Através das ações você pode fazer as mesmas verificações feitas pelo Notes e combiná-las para atingir outros resultados. Abaixo está uma lista com as ações e propriedades deste painel.

(ação) fixHTML
Tenta corrigir o código HTML do editor atual. O código HTML corrigido será retornado se o código orginal não tiver problemas que não podem ser corrigidos automaticamente. O painel exibirá os erros e avisos encontrados.

(ação) fixXHTML
O mesmo que fixHTML. Porém, o código de saída será XHTML e não HTML. Pode ser usado para converter documentos HTML em documentos XHTML.

(ação) fixXML
O mesmo que fixHTML. Não é possível converter um documento HTML para XML (use fixXHTML para isto), pois esta opção faz o HTMLTidy usar um parser XML e verificar qualquer incompatibilidade com as especificações do XML.
