Outras ferramentas


O menu executar pode executar qualquer programa, permitindo que você integre ao Notes as ferramentas que você usa. Vamos dar o exemplo de como configurar uma ferramenta no Notes, já que o processo é bastate semelhante para todas elas.

Integrando o CTAGS ao Notes

Uma das ferramentas muito usada pelos programadores em geral é o ctags, ferramenta que gera um arquivo com os símbolos de programação (funções, variáveis, classes) e a linha que eles ocorreram. Uma maneira de integrar o CTAGS ao Notes é usar o menu executar. Se você não conhece o ctags, vale à pena conhecê-lo: http://ctags.sf.net.

No menu executar, crie um novo item chamado "ctags" e preencha com os seguintes dados:
Executar: [Caminho para o CTAGS]\ctags.exe
Argumentos: -x "%file%"
Dir. Inicial: %path%
Depois escolha a opção "Capturar saída no painel saída" e clique no botão "ver opções". Nós vamos usar uma expressão regular para que o Notes interprete corretamente as "tags" produzidas pelo CTAGS, já que o parser automático do Notes é voltado apenas para compiladores/interpretadores. Desmarque a opção que diz para o Notes usar o parser automático e entre os seguinte dados:
Expressão regultar para interpretar a saída:
^[0-9a-zA-Z_\.]+[ \t]+[0-9a-zA-Z_\.]+[ \t]+([0-9]+)[ \t]+([a-zA-Z]:.+\.[0-9a-zA-Z_\.]+)[ \t]
Sub-expressão do arquivo: 2
Sub-expressão da linha: 1

Pronto, basta salvar e executar o item CTAGS com um arquivo de código C/C++, Java, Pascal, JavaScript, PHP, etc. e você terá um "code explorer".

A maior dificuldade para configurar uma ferramenta como o CTAGS é construir a expressão regular que irá interpretar a saída do erro, principalmente se você não tiver um bom entendimento sobre expressões regulares. No caso de você ter dificuldades, peça ajuda no fórum do Notes. Lembre-se de colocar um exemplo da saída da ferramenta para que as pessoas possam ajuda-lo.
