Adicionando suporte a novas linguagens


Para adicionar suporte a uma nova linguagem ao Notes você deve copiar o suporte de alguma das linguagems e renomear os arquivos para o nome da linguagem que você quer que o Notes entenda. O primeiro capítulo sobre cutomização ensina como funciona a estrutura dos tipos de arquivo e explica os conteitos básicos.

Depois você deve customizar a coloração e o SmartIndent. Por fim, você deve configurar o arquivo de opções como ensinado na próxima sessão.


O arquivo de opções

O arquivo de opções (o que leva o nome da linguagem e a extensão "xml") precisa ser editado a mão. Há nele algumas tags que não aparecem no diálogo de preferências, pois se referem a sintaxe da linguagem, ou seja, não são extamente opções.

Copie o arquivo de opções de alguma outra linguagem e abra ele no Notes. Eis as tags que você deve editar:

    LineComment - preencha com o comentário de linha mais usado na linguagem de programação que você está editando. Por exemplo, se fosse C/C++, preencheríamos com "//".

    MultilineCommentStart - Início de comentários para multiplas linhas mais usado na linguagem. Para C/C++ usaríamos "/*"; para pascal "{".

    MultilineCommentEnd - Final de comentários para multiplas linhas mais usado na linguagem. Para C/C++ usaríamos "*/"; para pascal "}".

    ScopeBegin - Início comum do escopo para variáveis e objetos da linguagem. O formato é uma lista de símbolos separados por espaços. Para C/C++ usaríamos "{"; para pascal "begin".

    ScopeEnd - Final comum do escopo para variáveis e objetos da linguagem. O formato é uma lista de símbolos separados por espaços. Para C/C++ usaríamos "}"; para pascal "end;".

É isto. Agora basta usar os diálogos do Notes para deixa o suporte para este tipo de arquivo bem completo!
